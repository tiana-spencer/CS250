How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?
- Collecting feedback is helpful when interpretting and implementing user needs because it is a way to gain insight to the user's experience with the program. From there you will have a better understanding of how it is used and the problems that users may encounter when using it. Creating user stories can help because they provide a simplified explanation of who uses the product, what features they would like to have with it, and how those features solve problems and/or add value within the product.
